feat(module): two-pass build with size_bytes, richer analyze, wipe-by-default

Add size_bytes column to nodes table; build() now runs two passes:
pass 1 extracts nodes/edges, pass 2 re-stats each file for byte sizes.

stats() extended with total_size_bytes and size_by_top_dir.
analyze() rewritten with summary table, ASCII bar chart (size by dir),
and Markdown tables for node/edge breakdowns.

build() default changed to wipe=True; CLI --wipe flag replaced by
--no-wipe. Schema now uses DROP/CREATE instead of CREATE IF NOT EXISTS
to prevent column-mismatch errors on schema evolution.

.gitignore switched from whole-directory KG exclusions to fine-grained
patterns that exclude DB/vectors but track snapshots/. Added .agentkg/.

Bump version 0.4.1 → 0.5.0.
